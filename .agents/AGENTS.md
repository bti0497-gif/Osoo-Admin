# 성적서 수동 매칭 UI/UX 보호 규칙

## 1. 개요
성적서(측정기록부) 수동 매칭 화면([ManualMatchingView.jsx](file:///e:/Wastewater%20Treatment%20Plant%20Admin/src/features/certificate/pdf-parser/components/ManualMatchingView.jsx))의 UI/UX는 원클릭 매칭 및 양방향 스와핑 리스트뷰 형태로 고안되어 고속의 처리 속도와 최적의 사용성을 제공합니다. 이 고급 구조가 함부로 단순화되거나 예전 드롭다운 방식으로 복원되지 않도록 보호합니다.

## 2. 레이아웃 설계 원칙
* **3열 구성 유지**:
  * **1열 (좌측)**: 페이지 목록 썸네일 리스트뷰 (`PageThumbnailPanel`)
  * **2열 (중앙)**:
    * 상단: 현장명 영역 크롭 뷰 (고정 높이 `110px`, 줌 및 제어 버튼 제거)
    * 하단: 남은 현장 리스트뷰 (실시간 필터 검색창 포함, 가나다 한국어 사전순 정렬 유지)
  * **3열 (우측)**:
    * 상단: 지정 완료 현장 리스트뷰 (세로 크기 `250px` 고정) + 되돌리기 버튼 (`← 되돌리기`, 한 항목 선택 시 활성화)
    * 하단: 날짜 선택 패널 (`DateSelector`) + 파일명 편집기 (`FileNameEditor`) + 매칭 최종 완료 버튼
* **높이 정렬 고정**: 1열, 2열, 3열의 하단 경계선이 완벽하게 일렬로 맞춰지도록 3열의 `overflow: 'hidden'` 및 `height: '100%'` 레이아웃 구조를 보존하고 임의로 스크롤로 밀어내지 마십시오.

## 3. 기능 및 데이터 흐름 원칙
* **원클릭 매칭**: 중앙 남은 현장 목록의 현장을 클릭하면 즉시 매칭이 지정되고, 다음 미지정 페이지로 자동 포커스가 이동해야 합니다.
* **양방향 스와핑 및 되돌리기**: 우측 지정 완료 리스트에서 한 항목을 선택하고 ` 되돌리기`를 누르면 매칭이 해제되어 미지정(남은 현장) 리스트로 롤백되어야 합니다.
* **불변성 상태 관리**: Set 객체(`availableSites`, `usedSites`) 변경 시 반드시 `new Set(prev)`과 같이 새로운 참조 주소를 만들어서 갱신해야 React가 실시간 변경 사항을 감지합니다. 뷰 단에서 Set의 직접 변조(`delete`, `add`)를 금합니다.
* **파일명 개별 연동 및 spellCheck 해제**: 우측 리스트에서 특정 매칭 행을 클릭하면 하단 파일명 편집 텍스트 박스에 해당 페이지 고유의 파일명이 연동되어 편집할 수 있어야 합니다. 텍스트 박스는 빨간 줄 방지를 위해 `spellCheck={false}`를 유지해야 합니다.
* **2자리 연도 날짜 파싱**: `YY.MM.DD` 형식의 파일명이 들어오더라도 `namingRules.js`는 `20YYMMDD`로 정상 파싱해내야 합니다.
* **완료 시 크롭 뷰 초기화**: 모든 페이지의 매칭이 끝났을 때는 줄 쳐진 임의의 잔상 이미지가 아닌 `모든 페이지의 현장 지정이 완료되었습니다.` 텍스트가 표시되도록 초기화 로직을 보존하십시오.

## 4. ROI 이미지 비동기 생성 및 상태 업데이트 제약 조건
* **이중 setPages 호출 및 레이스 컨디션 차단**:
  * `usePdfLoader.js` 내부의 `generateRoiImage` 함수는 비동기로 처리되며, 내부적으로 `setPages`를 호출하여 pages 상태에 `roiImage`를 직접 저장합니다.
  * 이에 따라, `ManualMatchingView.jsx` 등 호출 측 컴포넌트 내의 `useEffect`에서 `generateRoiImage`의 반환값을 받아 또다시 `setPages`를 실행하는 **이중 상태 업데이트 구조를 금지**합니다.
  * 이중 업데이트는 React의 상태 갱신 과정에서 이펙트 정리(cleanup: `active = false`)를 트리거하여 비동기식 크롭 이미지 로드를 영구히 누락시키는 경합 조건(Race Condition)을 유발하므로, 절대 되돌리거나 수정하지 마십시오.
* **불필요한 이펙트 실행 및 cleanup 방지**:
  * ROI 이미지 로딩 `useEffect`가 비동기로 진행되는 도중에 `pages` 배열 상태 자체가 의존성 변화에 의해 강제 리빌드 및 cleanup 되지 않도록, 상태 업데이트 참조가 꼬이지 않게 강력히 관리해야 합니다.
  * 디버깅 시에는 컴포넌트 내부에 심겨진 `[ManualMatchingView] loadRoiForCurrentPage` 디버그 로그의 진행 상태를 먼저 확인하십시오.

## 5. PDF 드롭 및 업로드 전송 마이크로 인터랙션 보호
* **드롭 즉시 로딩 오버레이 노출**:
  * PDF 파일을 드래그 앤 드롭했을 때, 전체 페이지 수 분석 완료 전이라도 드롭 즉시 로딩 화면을 띄워 중복 드롭을 막아야 합니다.
  * 조건문은 `(pdfProgress.total > 0 || pdfProgress.message !== '')` 형태를 엄격하게 보존하십시오.
* **업로드 중 중복 클릭 차단 및 로딩 시각화**:
  * 이미지 변환 및 드라이브 전송이 시작되면 완료 전(`isUploading === true` 일 때) 메인 버튼과 취소 버튼을 즉시 `disabled` 처리하여 API 중복 호출을 막아야 합니다.
  * 이 기간 동안 버튼 및 메시지 옆에 회전하는 `Loader2` 스피너를 보여주어야 하며, 마우스 커서는 `wait` 상태를 유지해야 합니다.
  * 업로드 진행률이 0%인 상태(극초기)라도 사용자에게 작업 중임을 인식시킬 수 있도록 진행바와 메시지 박스를 숨기지 말고 즉시 노출해야 합니다.
